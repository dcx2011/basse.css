What is basse.css?

Basse.css is a collection of CSS classes that can be used as a starting point for any web project. It's focused on the non-layout elements, and contains some tips to avoid the "mass-reproduction of classes".

The idea behind

Everybody knows that when you use a <h1> element you are going to have a bigger heading title than when you use an <h3>: it has a semantical meaning that is is generally translated to the design. basse.css tries to use that same logic with other elements like paragraphs or buttons. It is easy for a developer to remember that the biggest paragraph is <p class="p1"></p>. For example, the paragraph that you are reading is a p3, just bigger than normal, smaller than a p2.

As a convention, in basse.css the 'normal' size is 4. Everything below 4 (3,2,1) is 'big', and 5 or 6 are used for 'smaller' sizes. And then of course, you have the .big and .small classes that change the size of any inline element.

Combine them with care: you are risking to fall to the dark side of CSS hacks.

<b> is not a button and <i> is not an icon

True. But nowhere says that you can't break the rules. Anyway, we have decided that we we are going to use the 'classic' italic <i> element, now deprecated, for icons. The <b> element is also deprecated, but we are just going to steal a class for buttons (.b). As explained later, just don't use <b>. (By the way, this paragraph has a 'normal size' p4 class, but is unnecessary, so it doesn't exist in the CSS file).

Buttons

You can use a .b class for making a button from any element (for example, a link), but if you really need a button, we encourage you to use a <button> or <input type="submit" />. In this case a <b> has no semantical sense and won't work.
